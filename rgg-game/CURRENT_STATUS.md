# Актуальный статус проекта

Дата актуализации: 2026-05-17.

## Коротко

Проект готов к первому живому запуску в текущей клиентской архитектуре. Основные карточные flow, колесо, B-Shop/Gambling, дуэли, налоги, отражение, легендарки и уведомления стабилизированы. Последние проверки проходят:

```bash
npm test
npm run build
npm run lint
```

Статус проверок:

- `npm test` - проходит, 33 теста.
- `npm run build` - проходит.
- `npm run lint` - ошибок нет.
- Остаются старые 7 warnings по React hook dependencies / unused eslint-disable.

## Что стабилизировано

- **Решен «Карточный ад»:** Применение карт "Подкрутка" и "Мистер Фиш" переведено на атомарные транзакции Firestore. Исключены ситуации потери карт при одновременных кликах.
- **Окно реакции на колесе:** Внедрен 5-секундный таймер блокировки принятия результата для админа.
- **Визуальная история колеса:** Добавлена панель "Манипуляции", показывающая стек примененных карт с аватарами игроков.
- Визуальные броски кубиков для карточных эффектов `inv_008`, `inv_009`, `inv_016`.
- Blind selection для `inv_010` / `inv_011`: дубликаты карт не ломают React keys, пустая рука цели не списывает карту при прямом использовании.
- `inv_012` работает как ответная карта через `reflect_response`.
- `inv_013` проверяет монеты перед открытием выбора и не списывает карту при невозможном отражении.
- `inv_014` дуэль: отказ, Fish, выбор оружия, ставки, кубики, кастомный итог админом.
- `inv_015` налоги: очередь ответов, банк налогов, Fish, Reflect, Promo, gambling-ответ; владелец карты не блокируется fullscreen-экраном ожидания.
- `inv_017` и Fish у колеса: доступны после первого результата колеса; read-only колесо доступно из правого сайдбара.
- `inv_018` золотая карта: виртуальная карта в руке/панели, скидка в B-Shop, зачеркнутая старая цена.
- Легендарные карты: уникальные, не падают повторно, при получении сразу активируются и пишутся в логи.
- Gambling может выдать легендарку с небольшим шансом, если она еще не выиграна.
- Галерея артефактов и коллекция легенд показывают актуальное состояние: админ больше не видит все карты раскрытыми автоматически.
- Уведомления повторно не показываются после обновления страницы благодаря локальному списку просмотренных уведомлений.
- **Цветные логи:** Имена в логах автоматически раскрашиваются в цвета аур игроков.
- **Статус голосования:** Админ видит список игроков с иконками ✅/⏳ (кто проголосовал, а кто нет).
- **Алфавитный порядок:** Все списки участников теперь отсортированы по логину.
- **Чистота таблиц:** Из общей таблицы рейтинга удален избыточный столбец "Бонусы" (оставлен в истории раундов).
- Правая панель содержит коллекцию легенд, галерею артефактов и информационное колесо.
- В списке колеса поддерживается поле `url`; клик по игре открывает ссылку в новой вкладке.

## Firebase / правила

В живой базе сейчас используются более мягкие правила, где `gameState` открыт для записи авторизованным пользователям. Это рабочий компромисс для первого запуска, потому что большая часть логики остается клиентской.

Риски:

- Любой авторизованный игрок технически может записать некорректный `gameState`.
- Критичные операции лучше постепенно переносить в callable/backend handlers.
- После переноса можно будет сужать Firestore rules без риска сломать карты.

Первый кандидат для backend-миграции остается:

1. `playCard`
2. `respondToCard`
3. `resolveInteraction`

## Оставшиеся риски

- `useGameData.ts` все еще большой и остается главным источником потенциальных регрессий.
- `AppClean.tsx` все еще перегружен overlay-логикой.
- Некоторые старые hardcoded тексты еще не вынесены в `src/i18n/ru.ts`.
- Медиа стали легче, но часть GIF/WebP все еще крупные.
- Мобильная адаптация не завершена.
- Firestore writes на `notifications` и `gameEvents` могут расходовать quota при активной игре.
